# VARUNA

VARUNA is a cinematic fraud interception command center for hackathon demos. It simulates a real-time UPI and multi-bank mule-chain attack, detects graph-based fraud spread, predicts next-hop dissipation, recommends freeze actions, and generates an enforcement-ready case brief.

## Why this feels judge-ready

- Serious national command-center visual language instead of a generic student dashboard
- End-to-end demo story in one flow: fraud entry -> 3-hop mule expansion -> prediction -> freeze -> downloadable brief
- Guide-aligned demo workflow: inject fraud -> analyze account -> inspect chain -> freeze -> evidence dossier
- Explainable detection stack that works now and cleanly evolves into GAT + LSTM later
- Clean FastAPI services and React/Tailwind frontend for rapid iteration

## Stack

- Frontend: React, Vite, Tailwind CSS, Framer Motion, react-force-graph-2d
- Backend: FastAPI, Pydantic, in-memory scenario services, static report generation
- Backend flow: in-memory ingest, account-centric analysis, 3-hop chain extraction, mock freeze orchestration
- Data: deterministic seeded fraud scenario in memory plus `backend/seed_data.json`

## Modules delivered

### 1. VARUNA Pulse
- National overview metrics for suspicious chains, recoverability, dissipation, banks affected, and intercept time
- Live animated event feed
- Threat index and urgency framing for demo intensity

### 2. VARUNA Sentinel
- Live force-directed graph of the mule chain
- Rule-enhanced risk scoring for velocity, fan-out, burst timing, and multi-hop spread
- Account risk, chain confidence, human coordination score, and dissipation risk
- Clickable node drilldown with account-level investigation panel

### 3. VARUNA Intercept
- Multi-bank freeze action simulation
- Frozen and recommended accounts with freeze ETA
- "Simulate 3-minute delay" loss growth panel
- Account-specific freeze trigger path exposed via API and UI alert panel

### 4. VARUNA Brief
- HTML and JSON investigator package generated by the backend
- Includes summary, timeline, account-wise risk table, rationale, chain diagram narrative, and freeze actions

### 5. Predictive Dissipation Tracker
- Predicted next-hop accounts highlighted in orange on the graph
- Directly supports the story that VARUNA acts before funds fully disappear

## Detection logic

The current MVP uses an explainable scoring pipeline:

- Velocity score: how quickly an account relays funds onward
- Fan-out score: how many unique beneficiaries appear in rapid succession
- Burst score: coordinated timing and repeated split behavior
- Off-ramp awareness: wallet, crypto, and likely cash-out corridors raise dissipation risk

This is intentionally ML-ready:

- Graph features can become GAT node embeddings
- Transaction sequences can flow into an LSTM or temporal transformer
- The service boundary stays stable, so the frontend and API contract do not change
- The repo now includes explicit `GAT-placeholder` and `LSTM-placeholder` services so the call sites and scoring flow are demo-visible today

## Run locally

### Backend

```bash
cd backend
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --reload
```

API: [http://127.0.0.1:8000/api/dashboard](http://127.0.0.1:8000/api/dashboard)

### Frontend

```bash
cd frontend
npm install
npm run dev
```

App: [http://127.0.0.1:5173](http://127.0.0.1:5173)

Optional:

- Set `VITE_API_URL` if your backend is not at `http://127.0.0.1:8000/api/dashboard`

## Demo flow

1. Open the command center and start in national overview mode.
2. Hit `Inject Fraud Sequence` to trigger the scenario.
3. Point to the threat index, active chains, and recoverable amount.
4. Zoom into the Sentinel graph, click a mule node, and inspect the alert panel.
5. Call out the orange predicted next-hop nodes as the dissipation tracker.
6. Trigger a freeze action and show the recoverability story.
7. Close in the Evidence Dossier view and open the generated HTML action brief.
